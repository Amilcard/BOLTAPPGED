WORKFLOW — SAFE EDIT

Before any modification:
1. Explain what will be changed
2. Modify minimal code only
3. Verify nothing else broke
4. Confirm result

WORKFLOW — CONTENT EDIT

When editing a stay:
1. Check if content is partner source or GED edited
2. Modify GED version only
3. Protect from sync overwrite
4. Verify front display

WORKFLOW — BUG FIX

1. Identify exact location
2. Minimal fix
3. Verify routes and navigation
4. Confirm UI works

WORKFLOW — DATA CHECK

When using Supabase:
1. Verify fields exist
2. Avoid destructive update
3. Test with one record first
4. Confirm front coherence
